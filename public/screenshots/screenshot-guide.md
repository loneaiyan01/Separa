# Screenshot Guide for PWA

Screenshots enhance the install experience on some devices (Android especially).

## Required Screenshots:

### Mobile (Narrow Form Factor)
- **Size**: 540x720 pixels minimum
- **Aspect Ratio**: 3:4 recommended
- **Format**: PNG or JPEG
- **Filename**: `mobile-1.png`

### Desktop (Wide Form Factor)
- **Size**: 1920x1080 pixels minimum
- **Aspect Ratio**: 16:9 recommended
- **Format**: PNG or JPEG
- **Filename**: `desktop-1.png`

## How to Capture Screenshots:

### Method 1: Browser DevTools
1. Open Separa in Chrome/Edge
2. Press F12 to open DevTools
3. Click the device toolbar icon (Ctrl+Shift+M)
4. Select a mobile device (e.g., iPhone 12)
5. Navigate to a video room or lobby
6. Use Windows Snipping Tool or screenshot tool to capture

### Method 2: Real Device
1. Open Separa on your phone/tablet
2. Navigate to the main features (lobby, video room)
3. Take screenshots using device screenshot function
4. Transfer to computer

### Method 3: Figma/Mockup
1. Create mockups in Figma
2. Show the app interface in use
3. Export at the required sizes

## Tips:
- Show the app in action (video rooms, lobby, features)
- Use high-quality, clear screenshots
- Avoid sensitive information
- Show the key value proposition
- You can add up to 8 screenshots total

## Temporary Solution:
You can skip screenshots initially - they're optional but recommended for better install prompts.
